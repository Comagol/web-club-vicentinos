# Modelo de Datos — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a Prisma + PostgreSQL schema implementing the full data model from `docs/superpowers/specs/2026-06-25-modelo-datos-design.md`, with migrations and smoke tests proving every relation and constraint actually works against a real database.

**Architecture:** A standalone `backend/` Node.js + TypeScript project containing only the Prisma schema, migrations, and schema-level smoke tests. No HTTP server, no auth, no business-logic endpoints — those belong to chat2 (backend architecture), chat4/5 (frontend). This plan's only deliverable is a working, migrated, type-safe database schema. Postgres runs locally via Docker Compose for development/testing; Supabase hosting (per chat2) is a deployment concern, not part of this plan.

**Tech Stack:** Node.js, TypeScript, Prisma ORM, PostgreSQL 16 (Docker for local dev), Vitest for smoke tests.

## Global Constraints

- Database engine: PostgreSQL (decided in chat2).
- ORM: Prisma (decided in this session).
- No HTTP framework, no auth, no business logic in this plan — schema only.
- All enums and config tables must match exactly what's in `docs/superpowers/specs/2026-06-25-modelo-datos-design.md`.
- Every model gets a smoke test that writes and reads real rows through a real Postgres instance — no mocking the database.

```typescript
// backend/test/helpers/prisma.ts
import { PrismaClient } from "@prisma/client";

export const prisma = new PrismaClient();

// Order matters: each table must be deleted before any table it has a
// foreign key pointing to (topological sort of the FK graph). In particular
// PagoMercadoPago references both Cuota and Pedido, so it must be deleted
// before either of those, not after.
const TABLES_IN_DELETE_ORDER = [
  "UsuarioRol",
  "GrupoFamiliarMiembro",
  "SocioDivisionTemporada",
  "SubcomisionMiembro",
  "TarifaCuota",
  "MedioPagoAdherido",
  "ArchivoRendicion",
  "PagoMercadoPago",
  "ReservaEspacio",
  "InscripcionActividad",
  "TareaHistorial",
  "ItemPedido",
  "Noticia",
  "Resultado",
  "GrupoFamiliar",
  "Division",
  "Temporada",
  "ArchivoDebito",
  "Cuota",
  "Pedido",
  "Espacio",
  "ActividadRecaudatoria",
  "Tarea",
  "VarianteProducto",
  "Solicitud",
  "Empleado",
  "Producto",
  "Socio",
  "Subcomision",
  "Usuario",
  "CategoriaSocio"
];

export async function cleanDatabase(): Promise<void> {
  for (const table of TABLES_IN_DELETE_ORDER) {
    await prisma.$executeRawUnsafe(`DELETE FROM "${table}"`);
  }
}
```

This list is built up front against the final schema (Tasks 2–8 below) so later tasks don't need to keep editing this file. Deletes run child-tables-first to respect foreign keys.
